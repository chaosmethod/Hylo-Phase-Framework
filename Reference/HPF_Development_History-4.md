# 1\. What this document is for

This document records **how the framework actually developed**.

That matters because the present HPF package contains a mix of:

* empirical anchors,
* bottom-up discoveries,
* later structural closures,
* theorem-track corrections,
* milestone promotions.

If chronology is lost, the package becomes vulnerable to two bad readings:

1. **Retrodictive flattening** — the false idea that every result was derived in finished form from the start.
2. **Status confusion** — the false idea that because something was originally discovered empirically, it must still be theoretically open now.

This document prevents both mistakes.

It is historical by purpose, but it is updated so its status labels agree with the rewritten canon volumes and live support notes.
